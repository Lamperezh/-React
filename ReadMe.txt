Проєкт побудовано на React 18 + Vite, з використанням React Router v6 для маршрутизації. Є три маршрути:

головна сторінка
сторінка міста
сторінка з обраними містами

Дані про погоду отримую з OpenWeatherMap API (безкоштовний ключ).
Використовую async/await + Promise.all, щоб одночасно завантажувати поточну погоду та прогноз на 5 днів — це значно прискорює роботу додатка.
Глобальний стан реалізовано через Context API.
У контексті зберігаються:

обрані міста (зберігаються в LocalStorage)
історія пошуку (останні 5 міст)

Завдяки цьому дані залишаються навіть після перезавантаження сторінки.
Дизайн повністю адаптивний, створений на Tailwind CSS.
На мобільних пристроях картки розташовуються в колонку, на десктопі — гарною сіткою.
Є повна обробка всіх станів:

loading з красивим текстом
повідомлення про помилку, якщо місто не знайдено
повідомлення «Нічого не знайдено», якщо обраного немає

Також реалізовано:

валідацію форми пошуку
підйом стану
умовний рендеринг
захист від повторного додавання міста в обране

Тестування проведено за допомогою Vitest + React Testing Library:

тести на рендеринг компонентів
snapshot-тестування
всі тести проходять успішно


На головній сторінці вводимо назву будь-якого міста українською чи англійською — наприклад, «Київ», «Львів», «London».
Натискаємо «Пошук» — потрапляємо на сторінку міста, де бачимо поточну погоду та прогноз на 5 днів.
Можна додати місто в обране — зелена кнопка «Додати в обране», при повторному натисканні — видаляється.
У меню зверху є посилання «Обране» — там усі збережені міста, клікаємо — і одразу бачимо погоду.
Дані зберігаються навіть після закриття браузера — бо використовується LocalStorage.
